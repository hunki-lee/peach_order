# 복숭아 주문서 정적 웹앱

카카오톡으로 복숭아 주문을 받을 때 고객이 표준 주문 문구를 쉽게 만들 수 있는 정적 웹앱입니다. 서버 저장 없이 브라우저 안에서만 입력값을 처리합니다.

## 파일 구조

```text
.
├── index.html
├── styles.css
├── script.js
└── README.md
```

- `index.html`: 주문 입력 폼과 생성된 주문 문구 영역
- `styles.css`: 모바일 중심 화면 스타일
- `script.js`: 주소 검색, 주문 문구 생성, 필수값 확인, 클립보드 복사
- `README.md`: 배포 및 운영 안내

## 사용 방법

1. 고객이 주문 정보를 입력합니다.
2. 받는 사람 주소는 `주소 검색` 버튼으로 찾고 상세주소를 입력합니다.
3. 여러 곳으로 보낼 때는 `+ 받는 사람 추가` 버튼으로 수취인을 추가합니다.
4. 아래쪽 또는 오른쪽의 주문 문구를 확인합니다.
5. `복사하기` 버튼을 누릅니다.
6. 부모님 카카오톡방에 붙여넣습니다.
7. 입금계좌가 필요하면 `계좌 복사` 버튼으로 계좌만 따로 복사합니다.

입력값은 서버로 전송되지 않으며, 현재 브라우저 화면 안에서만 사용됩니다.

생성되는 문구는 카카오톡에 붙여넣기 쉽도록 보내는 사람, 받는 사람, 주소, 상품 정보만 짧게 정리합니다. 입금계좌는 주문 문구 아래에 따로 표시하고 별도 버튼으로 복사합니다.

## 주소 검색

주소 검색은 Kakao/Daum 우편번호 서비스를 사용합니다. 페이지 자체는 정적 파일로 동작하지만, 주소 검색 버튼을 누르는 순간 외부 주소 검색 스크립트를 불러오므로 인터넷 연결이 필요합니다.

사용한 스크립트 주소:

```text
https://t1.daumcdn.net/mapjsapi/bundle/postcode/prod/postcode.v2.js
```

## 로컬에서 확인하기

별도 빌드 과정이 없습니다. `index.html` 파일을 브라우저로 열면 됩니다.

간단한 로컬 서버로 확인하려면 아래 명령을 사용할 수 있습니다.

```bash
python3 -m http.server 8000
```

그다음 브라우저에서 `http://localhost:8000`으로 접속합니다.

## GitHub Pages 배포

1. GitHub에서 새 저장소를 만듭니다.
2. 이 폴더의 파일을 저장소 루트에 올립니다.
3. GitHub 저장소 화면에서 `Settings`로 이동합니다.
4. 왼쪽 메뉴에서 `Pages`를 선택합니다.
5. `Build and deployment`의 Source를 `Deploy from a branch`로 선택합니다.
6. Branch를 `main`, 폴더를 `/root`로 선택하고 저장합니다.
7. 잠시 뒤 표시되는 Pages 주소로 접속합니다.

## 운영 메모

- 서버 저장 기능이 없으므로 주문 최종 접수는 카카오톡 메시지를 기준으로 관리합니다.
- 카카오톡 인앱 브라우저에서 복사가 막히는 경우가 있으면, 생성된 주문 문구를 길게 눌러 직접 선택 후 복사하면 됩니다.
- 주소 검색이 열리지 않으면 인터넷 연결을 확인한 뒤 다시 시도합니다.
- 품종 목록은 `index.html`의 `<select id="variety">` 옵션을 수정해 바꿀 수 있습니다.
- 입금계좌는 `script.js`의 `BANK_ACCOUNT` 값을 수정해 바꿀 수 있습니다.
